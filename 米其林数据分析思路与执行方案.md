# 米其林数据分析思路与执行方案

## 1. 你的两个目标
1. 快速看结果：先做基础统计、透视和图表。
2. 形成洞察：解释城市差异、菜系结构、中西餐分布和潜在原因。

本方案基于总表 CSV：
- download/michelin_china_restaurants_master.csv

## 2. 可以分析哪些问题（按价值排序）

### 2.1 菜系占比
- 中国米其林中，哪些菜系出现频率最高？
- Top 10 菜系占比是否过高（集中度）？

指标：
- 菜系出现次数
- 菜系占比
- HHI 或 Top-N 占比（可衡量集中度）

### 2.2 各城市差异
- 每个城市的餐厅总量、星级结构、均价带（price）分布。
- 某些菜系是否明显集中在某些城市（如粤菜/淮扬/法餐）。

指标：
- city x michelin_award 透视
- city x cuisine 透视
- 城市菜系多样性（香农指数）

### 2.3 中国有哪些美食（米其林视角）
- 从“出现次数 + 覆盖城市数”识别全国性强势菜系。
- 按大类归并（中餐地方菜系、日韩、东南亚、西餐等）形成更高层结论。

指标：
- 菜系覆盖城市数
- 菜系平均星级结构

### 2.4 中国西餐有哪些
- 先定义西餐口径：法餐、意大利菜、西班牙菜、欧陆、美式牛排等。
- 统计西餐在各城市的占比、星级分布、价格带分布。

指标：
- western_flag（是否西餐）
- city x western_flag
- western x award / price

## 3. 两种执行方式：写代码 vs 直接给在线 AI

## 3.1 方式 A：写代码分析（推荐主路径）
适合：
- 要可复现、可迭代、可审计。
- 需要多轮探索、反复改口径。

优点：
- 结果可重复。
- 口径可控（尤其菜系归并规则）。
- 后续可自动化更新。

建议工具：
- pandas：清洗和透视
- matplotlib/seaborn：可视化
- jupyter notebook：交互探索

最小示例：
```python
import pandas as pd

df = pd.read_csv("download/michelin_china_restaurants_master.csv")

# 1) 菜系列拆分
s = (
    df.assign(cuisine=df["cuisines"].fillna("").str.split("; "))
      .explode("cuisine")
)
s = s[s["cuisine"].str.len() > 0]

# 2) 菜系占比
cuisine_share = (
    s["cuisine"].value_counts(dropna=False)
      .rename_axis("cuisine")
      .reset_index(name="count")
)
cuisine_share["share"] = cuisine_share["count"] / cuisine_share["count"].sum()

# 3) 城市 x 星级
city_award = pd.crosstab(df["city"], df["michelin_award"])

# 4) 城市 x 菜系（Top）
city_cuisine = pd.crosstab(s["city"], s["cuisine"])

print(cuisine_share.head(15))
print(city_award)
```

## 3.2 方式 B：直接把 CSV 发给在线 AI
适合：
- 想快速拿到描述性结论。
- 一次性问题，不追求长期复用。

优点：
- 上手快，问答效率高。

风险：
- 结果不可完全复现。
- 口径可能漂移（不同轮次说法不一致）。
- 隐私与合规需确认（上传前脱敏）。

最佳实践：
- 先让代码做“基准表”和“口径字段”（例如 western_flag、cuisine_group）。
- 再把处理后的 CSV 提供给在线 AI 做解释和摘要。

结论：
- 推荐“代码产出可信数据 + AI 做解释”的组合。

## 4. 分析口径设计（非常关键）

### 4.1 菜系字段处理
原始字段是多值字符串（以 `; ` 分隔）。建议：
1. explode 拆成一行一个菜系。
2. 统一别名（例如 "French"、"法餐" 归并）。
3. 建立 cuisine_group（中餐地方菜、泛亚洲、西餐、其他）。

### 4.2 西餐定义（建议版）
可先按关键词映射：
- French / 法餐
- Italian / 意大利菜
- Spanish / 西班牙菜
- European / 欧陆
- Steakhouse / 牛排馆
- Contemporary / 现代欧陆（视业务定义）

生成字段：
- western_flag: 0/1
- western_type: 法餐/意餐/欧陆/其他西餐

### 4.3 城市对比口径
至少固定三类指标：
1. 规模：餐厅数量
2. 质量层级：奖项结构（1星/2星/3星/Bib/Selected）
3. 结构：菜系分布与多样性

## 5. 推荐输出物（你可直接对外汇报）
1. 总览表：全国餐厅总数、城市数、主要奖项占比。
2. 城市排名表：按餐厅数、星级密度排序。
3. 菜系 Top 表：出现次数、占比、覆盖城市数。
4. 西餐专题表：各城市西餐数量、占比、奖项结构。
5. 一页图表：
   - 菜系 Top 10 条形图
   - 城市 x 奖项堆叠图
   - 城市 x 菜系热力图

## 6. 推荐执行流程（效率最高）
1. 先跑脚本生成最新 CSV。
2. 用 pandas 做口径字段（explode、归并、western_flag）。
3. 导出 3-5 张基础透视表。
4. 再交给 AI 做“解释层”输出（洞察、原因、策略建议）。
5. 对关键结论回到数据表做反查确认。
